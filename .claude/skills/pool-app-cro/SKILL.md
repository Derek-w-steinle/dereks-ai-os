# Pool inspection app — conversion (CRO)

Improve conversion for the pool inspection product. Start from evidence, then
propose specific, testable changes.

## Read shared context first

Read `docs/product-marketing.md` for audience and positioning, and apply
`derek-voice` to any copy you write.

## Audit before redesigning

1. Identify the one primary action of the screen.
2. List the friction: unclear value, too many fields, missing proof, weak CTA.
3. Map the path from landing to activated user and find the biggest drop-off.

## Levers, in priority order

- **Clarity** — one obvious value proposition above the fold.
- **Friction** — remove or defer every non-essential form field.
- **Proof** — add specific outcomes, not generic testimonials.
- **Action** — one primary CTA with concrete, benefit-led label copy.

## Output

Return a prioritized list: the change, the rationale, and how to measure it (the
metric and the test). Prefer one well-specified A/B test over many vague tweaks.

For brand voice, see `derek-voice`. For shared positioning, see
`product-marketing`.
